# REQ-8: Agent Definition File Line Limit

## Requirement

**All files in the `.agents/` directory must not exceed 750 lines (hard limit).** Files exceeding 500 lines should be flagged for structural evaluation (review threshold). This is a two-tier limit:

- **500 lines (review threshold):** Flag as a SUGGESTED finding. The file author should evaluate whether the file can be restructured, but no change is required if the file is well-structured at its current length.
- **750 lines (hard limit):** Flag as a REQUIRED finding. The file must be split or reduced before the gate can advance.

## Applicability

**This requirement only applies when a session modifies files in `.agents/`.** If no `.agents/` files are created or modified in a session, this requirement is not enforced and should not appear in compliance checks or gate findings.

## Rationale

```
Why a two-tier limit for .agents/ files?
----------------------------------------------------------------------
Different from code:  Agent definition files (pipelines, roles, shared
                      context, requirements) are functionally closer to
                      configuration than source code. A pipeline with
                      inline templates or multiple gates naturally runs
                      longer than a code module. The 500-line code limit
                      (REQ-3) does not account for this.

Early signal:         The 500-line threshold provides pressure to
                      evaluate structure before a file becomes
                      unmanageable. This caught SKILLGEN.md at 582 lines
                      and prompted evaluation -- the file was
                      well-structured and did not need splitting.

Hard boundary:        The 750-line hard limit prevents unbounded growth.
                      No .agents/ file should exceed this without being
                      restructured into smaller, focused components.

Precedent:            This mirrors the cyclomatic complexity thresholds
                      in the Quality Engineer's gate: flag at threshold,
                      require action at the hard limit.
----------------------------------------------------------------------
```

## Scope

- **Applies to:** All files under `.agents/` -- pipelines, roles, shared context, requirements, and any future file categories
- **Only when:** The session creates or modifies files in `.agents/`
- **Measurement:** Total lines including blank lines, frontmatter, and code blocks
- **Excludes:** Sessions that do not touch `.agents/` files -- this requirement is invisible in those sessions

## Enforcement Rules

```
This requirement is ONLY enforced when the session modifies .agents/ files.
If no .agents/ files are modified, skip REQ-8 compliance checks entirely.

Gate 4 (Engineer):     Check line counts of any .agents/ files created
                       or modified. Files over 500 lines: note in
                       implementation report. Files over 750 lines:
                       must split or reduce before advancing.

Gate 5 (Code           Any .agents/ file over 750 lines is a REQUIRED
Reviewer):             change. Files between 500-750 lines are a
                       SUGGESTED change (evaluate structure).

Gate 6 (Quality        Any .agents/ file over 750 lines not caught at
Engineer):             Gate 5 is a REQUIRED change. Files between
                       500-750 are a SUGGESTED change.
```
